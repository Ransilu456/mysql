Indexing — B-Tree Internals

An index is a sorted data structure that lets MySQL find rows without scanning the entire table.

indexing.sql
-- Single column index
CREATE INDEX idx_email ON users(email);

-- Composite index (left-prefix rule applies)
CREATE INDEX idx_name_age ON users(last_name, first_name, age);

-- Unique index
CREATE UNIQUE INDEX idx_sku ON products(sku);

-- Full-text index
CREATE FULLTEXT INDEX idx_body ON posts(body);

-- Covering index: query only uses index, never touches table
CREATE INDEX idx_cov ON orders(user_id, status, total);
SELECT total FROM orders WHERE user_id=1 AND status='paid';
-- Extra: Using index ← never touched the table!

-- Show existing indexes
SHOW INDEX FROM users;

-- Remove unused index
DROP INDEX idx_email ON users;

EXPLAIN — Reading Query Plans

EXPLAIN is the most important diagnostic tool in MySQL. It shows you exactly how the server will execute your query.

explain.sql
EXPLAIN SELECT u.name, o.total
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE u.role = 'admin' AND o.status = 'paid';

EXPLAIN Output Columns

transactions.sql
START TRANSACTION;

UPDATE accounts SET balance = balance - 500 WHERE id = 1;
UPDATE accounts SET balance = balance + 500 WHERE id = 2;

-- Only commit if both succeed
COMMIT;

-- Undo everything if error occurs
ROLLBACK;

-- Savepoints for partial rollback
SAVEPOINT after_debit;
ROLLBACK TO after_debit;
