Data Type Logic

Choosing the wrong data type wastes storage, kills index performance, and causes silent data corruption. Use this reference every time you design a column.

Numeric Types

TypeStorageRangeUse When
TINYINT1 byte-128 to 127Status flags, booleans
SMALLINT2 bytes-32,768 to 32,767Age, year fields
INT4 bytes±2.1 billionIDs, counters
BIGINT8 bytes±9.2 quintillionLarge IDs, timestamps
DECIMAL(p,s)VariableExact precisionMoney — NEVER use FLOAT
FLOAT4 bytesApproximateScientific data only
Common Mistake
wrong.sql
price FLOAT  -- ❌ 0.1 + 0.2 = 0.30000000004
correct.sql
price DECIMAL(10,2)  -- ✅ Exact: 0.10 + 0.20 = 0.30

CREATE TABLE products (
  id        INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

DDL Commands: Building Structure

DDL (Data Definition Language) commands define your schema. These do not modify data — they modify the structure that holds data.

CREATE TABLE — Full Syntax

create_table.sql
CREATE TABLE users (
  id         INT UNSIGNED AUTO_INCREMENT,
  username   VARCHAR(50)  NOT NULL UNIQUE,
  email      VARCHAR(255) NOT NULL UNIQUE,
  password   CHAR(60)     NOT NULL,
  role       ENUM('admin','user','guest') DEFAULT 'user',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id)
) ENGINE=InnoDB CHARSET=utf8mb4;

ALTER TABLE — Modify Existing Tables

alter_table.sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20) AFTER email;
ALTER TABLE users MODIFY COLUMN phone VARCHAR(30) NOT NULL;
ALTER TABLE users DROP COLUMN phone;
ALTER TABLE users RENAME TO members;
ALTER TABLE users ADD INDEX idx_email (email);
ALTER TABLE users DROP INDEX idx_email;

Keys & Constraints

Constraints are rules enforced by the database engine at the storage level — not at the application level. They are your last line of defense against corrupt data.

ConstraintWhat It DoesExample
PRIMARY KEYUnique, non-null row identifierid INT PRIMARY KEY
UNIQUENo duplicate values allowedemail VARCHAR UNIQUE
NOT NULLColumn must always have a valuename VARCHAR NOT NULL
DEFAULTFallback value if none givenstatus DEFAULT 'active'
CHECKCustom condition validationCHECK (age >= 18)
FOREIGN KEYReferential integrity between tablesREFERENCES users(id)
foreign_key.sql
CREATE TABLE orders (
  id         INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id    INT UNSIGNED NOT NULL,
  total      DECIMAL(10,2) NOT NULL,
  status     ENUM('pending','paid','cancelled') DEFAULT 'pending',
  CONSTRAINT fk_order_user
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE
);
