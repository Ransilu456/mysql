Normalization

1st Normal Form (1NF)

Rule: Every column must hold exactly ONE atomic value. No comma-separated lists, no arrays, no repeating groups.

BEFORE — Violates 1NF
IDNamePhones
1Sara555-1234, 555-5678
2John555-9999

Two phones in one cell — cannot index or query individually.

AFTER — 1NF Compliant
IDNamePhone
1Sara555-1234
1Sara555-5678
2John555-9999
1nf_schema.sql
CREATE TABLE user_phones (
  user_id  INT NOT NULL,
  phone    VARCHAR(20) NOT NULL,
  PRIMARY KEY (user_id, phone),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

2nd Normal Form (2NF)

Rule: Must be in 1NF. Every non-key column must depend on the entire composite primary key — not just part of it.

Normalization

BCNF — Boyce-Codd Normal Form

Rule: For every functional dependency X → Y, X must be a superkey. This is a stricter version of 3NF for tables with overlapping candidate keys.

3NF but NOT BCNF

A teacher only teaches one subject (Teacher → Subject), but Teacher is not a superkey.
